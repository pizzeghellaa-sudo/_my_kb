## Gap 1: Building Applications with the Claude API (Structural)

The wiki comprehensively covers *using* Claude Code as a developer tool but has almost nothing on *building* AI-powered applications with Claude as the model.

**What's missing:**
- Anthropic SDK / API mechanics: tool use, structured outputs, system prompts, streaming
- How to implement the 5 agent patterns (chaining, routing, parallelization, orchestrator-workers, evaluator-optimizer) in actual code
- Evaluation, testing, and reliability strategies for agent systems
- The "Building Effective AI Agents" source was captured at ~10% depth — the Anthropic engineering article is thousands of words; the wiki has ~400 [Source: ai-agent-patterns]

**Why it matters:** Someone wanting to build something with Claude as the AI brain hits a wall. The patterns are named but unimplemented.

**Fix:** Re-ingest `Building Effective AI Agents - Anthropic.url` via Playwright for full depth. Add sources on Anthropic SDK usage, tool use, and agent evaluation.

## Gap 2: Claude Code Practical Workflows (Structural)

The wiki covers *configuring* Claude Code thoroughly but not *operating* it effectively.

**What's missing:**

- When to use plan mode (`ccp`) vs regular mode — noted in [[claude-code-aliases]] but never elaborated
- The debugging loop: "let Claude iterate" is mentioned but not explained [Source: claude-code-fullstack]
- ~~Sub-agents — mentioned in [[claude-code-memory-auto]] but never documented~~ **FILLED:** [[claude-code-agents]] now covers `.claude/agents/` subagent personas (2026-04-07 ingest)

- End-to-end workflow: from task description to merged PR

**Why it matters:** You could read every page in this wiki and know exactly how Claude Code is configured but not know how to use it to build a feature.

**Fix:** Re-ingest `Claude Code for Fullstack Development` via Playwright for full depth. Add sources specifically on Claude Code workflows, plan mode usage, and sub-agent orchestration.
